# NCSKit Scales Module Audit & Upgrade Plan

## 1. Current Issues Assessment
- **Layout**: The desktop layout uses a modal for details which can feel "heavy". On mobile, the modal often overflows or becomes difficult to navigate.
- **Content**: Each scale only has basic info. Researchers need to know *how* to use it, *how* to score it, and if there are any specific conditions (e.g., reverse coding).
- **Mobile Experience**: Long lists of scale items are hard to read on a small screen. Tables don't fit well.

## 2. Proposed UI Enhancements
### Desktop
- **Catalog View**: Use a more dense grid with better category chips.
- **Detail Page**: Move away from modals to dedicated detail pages (or a modern "Side Panel" approach).
- **Interactive Items**: Allow users to "test" the items or see real-world examples.

### Mobile
- **Bottom Sheets**: For quick actions like "Copy Citation".
- **Accordion Groups**: For scale dimensions to reduce vertical scrolling.
- **Sticky Actions**: "Analyze Now" button should always be accessible.

## 3. Proposed Content Enhancements (Schema Expansion)
Add the following fields:
- `usage_guide`: Comprehensive instructions on how to administer the scale.
- `scoring_logic`: How to calculate the final scores (Average, Sum, Weighting).
- `reverse_items`: List of item codes that require reverse coding.

## 4. Technical Strategy
- Implement a `TabbedInterface` in `app/scales/[id]/page.tsx`.
- Use `framer-motion` for smooth layout transitions between categories.
- Enhance the Admin CMS to support these new fields.
